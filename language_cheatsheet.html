🛡️ Phishly Dev Cheat Sheet

HTML · CSS · JavaScript — for developers with C / Python experience

🧱 HTML — Structure of the Page

Think of HTML as declaring data types and containers. It defines what is on the page, not how it looks.

Basic File Structure

<!DOCTYPE html>                    <!-- tells browser this is HTML5 -->
<html lang="en">
  <head>                             <!-- metadata, not visible -->
    <meta charset="UTF-8">
    <title>Phishly</title>
    <link rel="stylesheet" href="popup.css">
  </head>
  <body>                             <!-- visible content goes here -->
    <h1>Hello</h1>
    <script src="popup.js"></script>  <!-- JS at bottom of body -->
  </body>
</html>

Common Tags

TagPurpose

<span>Inline container
<h1>–<h6>Headings
<p>Paragraph
<button>Clickable button
<textarea>Multi-line text input
<input>Single-line input / checkbox
<label>Label for an input
<ul> / <li>Unordered list / item

Attributes

Attributes go inside the opening tag, like named parameters:

<!-- id = unique identifier, like a variable name -->
<div id="result-box"></div>

<!-- class = reusable style group -->
<p class="warning-text">Be careful</p>

<!-- checkbox input -->
<input type="checkbox" id="asks-login">
<label for="asks-login">Asks me to log in</label>
💡 for Python devs: HTML tags are like function calls that wrap content. Attributes are keyword arguments. Closing tags (</div>) are required — no indentation-based scoping like Python.

🎨 CSS — Appearance and Layout

CSS selects elements and applies style rules. Think of it as a big set of conditional property assignments.

Syntax

/* selector { property: value; } */

h1 {
  color: #e74c3c;          /* hex color */
  font-size: 1.5rem;      /* rem = relative to root font size */
  margin-bottom: 12px;   /* pixel spacing */
}

.card {               /* .name = class selector */
  background: #1e293b;
  border-radius: 8px;
  padding: 16px;
}

#result-box {         /* #name = ID selector (unique element) */
  display: none;        /* hidden by default */
}

The Box Model

Every element is a box with layers outward:

/* content → padding → border → margin */

.box {
  width: 200px;         /* content width */
  padding: 10px;        /* space inside border */
  border: 1px solid #ccc;
  margin: 20px;         /* space outside border */
}

Flexbox (most useful for layout)

.container {
  display: flex;
  flex-direction: row;    /* or column */
  justify-content: space-between; /* horizontal */
  align-items: center;   /* vertical */
  gap: 12px;             /* space between children */
}

💡 for C devs: CSS has no variables by default unless you use var(--name) custom properties. Cascade = later rules override earlier ones. Specificity = more specific selectors win.

⚙️ JavaScript — Behavior and Logic

JS runs in the browser and manipulates the HTML/CSS dynamically. Syntax is C-like but loosely typed like Python.

Variables

let x = 10;          // mutable (use this usually)
const name = "phishly"; // immutable reference
// var exists but avoid it (old, scoping issues)

Types

let num   = 42;
let str   = "hello";
let bool  = true;
let arr   = [1, 2, 3];       // like Python list
let obj   = { key: "val" };  // like Python dict
let empty = null;
let undef;                  // undefined (not set yet)

Functions

// Traditional (like C)
function add(a, b) {
  return a + b;
}

// Arrow function (like Python lambda, but multi-line)
const multiply = (a, b) => {
  return a * b;
};

// Short arrow (single expression)
const double = x => x * 2;

Control Flow

// Same as C
if (score > 70) {
  console.log("high risk");
} else if (score > 40) {
  console.log("moderate");
} else {
  console.log("low");
}

// for loop (C-style)
for (let i = 0; i < arr.length; i++) { }

// for...of (like Python for x in list)
for (const item of arr) { }

Accessing & Modifying HTML (the DOM)

The DOM (Document Object Model) is how JS reads and changes your HTML. This is the most important concept for your plugin:

// Get elements
const box   = document.getElementById("result-box");   // by id
const items = document.querySelectorAll(".flag-item"); // by class, returns list
const btn   = document.querySelector("#analyze-btn");  // first match

// Read input values
const text = document.getElementById("email-body").value;

// Check checkbox
const checked = document.getElementById("asks-login").checked; // true/false

// Change content
box.textContent = "Analysis complete";     // safe: plain text only
box.innerHTML   = "<b>High Risk</b>";      // renders HTML (careful with user input!)

// Change style dynamically
box.style.display = "block";   // show element
box.style.display = "none";    // hide element
box.style.color   = "#ef4444";

// Add / remove CSS classes
box.classList.add("high-risk");
box.classList.remove("loading");
box.classList.toggle("visible");

Event Listeners — Responding to User Actions

// "When the user clicks #analyze-btn, run this function"
document.getElementById("analyze-btn").addEventListener("click", () => {
  // your analysis logic here
  console.log("Button clicked!");
});

// Other common events:
// "input"   → fires as user types
// "change"  → fires when checkbox is toggled
// "keydown" → fires when a key is pressed

String Methods (useful for Phishly's text analysis)

const text = "URGENT: Verify your account NOW!";

text.toLowerCase();           // "urgent: verify your account now!"
text.includes("urgent");      // false (case sensitive) → use toLowerCase first
text.toLowerCase().includes("urgent"); // true
text.split(" ");              // splits into array of words
text.trim();                   // removes leading/trailing whitespace
text.replace("NOW", "today"); // replaces first match

// Template literals (like Python f-strings)
const score = 72;
const msg = `Risk score: ${score}/100`; // backtick + ${}
⚠️ JS has no types: "5" + 3 = "53" (string concat), not 8. Use parseInt() or Number() when converting. Always use === (strict equality) not ==.

🧩 Chrome Extension Essentials

manifest.json (required)

{
  "manifest_version": 3,
  "name": "Phishly",
  "version": "1.0",
  "description": "Phishing risk educator",
  "action": {
    "default_popup": "popup.html",
    "default_icon": "icons/icon.png"
  },
  "permissions": []   // add "storage" here if you ever need it
}

How the 3 files connect

popup.html  →  loads popup.css (style)  +  popup.js (logic)
popup.js    →  reads from HTML inputs, writes to HTML outputs
popup.css   →  controls colors, layout, sizing
💡 Chrome extension popups are just HTML pages. No server, no backend. Everything runs locally in the browser — perfect for Phishly's privacy-first design.
